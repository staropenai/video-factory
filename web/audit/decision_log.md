| Dimension | New standalone matcher | Adapter over retrieveFromLocal |

| Rollback | 3 | 5 |

| Dimension | Direct Sentry calls everywhere | Thin wrapper module |

**Reason**: Centralized DSN check in `isSentryActive()` prevents crashes when Sentry is unconfigured. Single mock point for tests. Uses `metrics.distribution()` API (Sentry v10 recommended) instead of deprecated transactions.

## T4: Quota — Idempotency Approach

**Time**: 2026-04-13
**Selected**: SET NX + INCR pipeline (two-step, not Lua)

| Dimension | Lua atomic script | SET NX + INCR pipeline |
|-----------|------------------|----------------------|
| Correctness | 5 (true atomic) | 4 (race window between SET and INCR) |
| Change scope | 3 (Lua string) | 5 (uses existing pipeline pattern) |
| Rollback | 3 | 5 |
| Testability | 3 | 5 (standard mock) |
| Arch compat | 3 (Upstash REST may not support EVAL) | 5 (Upstash REST native) |

**Reason**: Upstash REST API doesn't support `EVAL` for Lua scripts natively. SET NX for idempotency key + existing INCR pipeline for quota is compatible with Upstash. The race window between SET and INCR is negligible at current request rates. DECR rollback handles over-count.
