## 设计哲学：三句话

李笑来在《自学是门手艺》开篇写道：

> `def teach_yourself(anything):`  
> `  while not create(): learn(); practice()`  
> `  return teach_yourself(another)`

这个伪代码说明一件事：**学习是循环，不是线性的。每轮循环都必须有产出（create），否则循环不闭合。**

对 JTG 项目，这句话翻译成工程语言就是：

1. **每次迭代必须产出可复用资产**（规则、快照、踩坑记录），否则经验流失
2. **完整比快速更重要**（李笑来：「小，无所谓；完整，才是关键」）
3. **规则必须机器可判断**，否则只是人类自我安慰

## 0. V2 与 V1 的本质差异

| 维度 | V1 | V2 |
|---|---|---|
| 定位 | 给人看的协作说明 | 机器可解析的执行协议 |
| 约束表达 | 自然语言段落 | YAML 规则对象，可 grep |
| 完成标准 | 「差不多完成」 | 唯一完成信号，脚本可验证 |
| 失败处理 | 建议记录 | 触发即强制写入 pitfall |
| 经验积累 | 事后补充 | 任务完成条件的一部分 |
| 冲突处理 | 未定义 | 明确优先级 + 升级路径 |

## 1. 执行权限层级

```
immutable_principles (IP)   ← 绝不触碰
       ↓ 覆盖
locked_requirements (LR)    ← 需明确授权才能变更
       ↓ 覆盖
iterable_parts (IT)         ← 可优化，需说明原因
       ↓ 覆盖
task_instruction            ← 当次任务指令
       ↓ 覆盖
style_preference            ← 偏好，最低优先级
```

**任何未被显式标记为可迭代的内容，默认不可改。**

## 5. 待确认部分（Pending）— 默认阻断执行

满足以下任一条件，自动进入待确认，**不得直接执行**：

- 可能影响 IP 或 LR 层
- 缺少唯一完成信号
- 单任务影响文件超过 3 个
- 引入新的外部依赖
- 修改用户可见文案但无来源证明
- 不确定是否可改

**阻断后必须输出：** 歧义描述 + 选项列表 + 推荐默认 + 等待确认

## 6. 证据门标准

> 「没有证据，不得改动。」

### 有效证据必须满足三条：
1. **可复现**：他人按同样步骤能得到同样结果
2. **可独立验证**：不依赖执行者记忆
3. **可定位**：有文件路径 / 命令输出 / 路由地址

### 有效证据形式：
```
grep 结果、文件路径+片段、命令输出、
路由状态码、截图引用、测试报告
```

### 无效证据（不接受）：
```
「应该在这里」「上次似乎修过」「我记得是这样」
主观推断、会话记忆、无法复现的描述
```

## 8. 任务完成的四个必要条件

> 缺任一项，视为未完成：

```
□ 1. 证据门 OPEN（先确认现状）
□ 2. 完成信号达成（脚本/grep/路由可验证）
□ 3. 审计无 P0 问题
□ 4. 若失败，已写入 pitfall 记录
```

## 9. 变更指令关键词

| 关键词 | 效果 |
|---|---|
| `升级固定原则` | 可修改 IP，修改前复述内容请确认 |
| `变更固定需求` | 可修改 LR，修改前复述内容请确认 |
| `优化可迭代部分` | 直接优化 IT，说明改动原因 |
| `仅讨论，不执行` | 只分析，不输出代码或指令 |
| `按现状实现` | 不优化，严格按要求完成 |

## 10. Claude Code 专属规则（CC）

| ID | 规则 |
|---|---|
| CC-1 | 每个任务必须先跑证据门，GATE: OPEN 才能继续 |
| CC-2 | 每个任务有且仅有一个完成信号（可脚本验证）|
| CC-3 | 修改已有文件前，先输出影响文件清单 |
| CC-4 | 单任务影响文件 ≤ 3 个；超过则拆分 |
| CC-5 | 同一任务不得同时修改 IP 层和 IT 层内容 |
| CC-6 | 任务结束必须运行验证脚本，不允许跳过 |
| CC-7 | 验证失败必须自动生成 pitfall 记录 |
| CC-8 | 发现待确认项必须阻断执行并输出选项 |

---

## 11. 冲突处理

```
IP 与 LR 冲突 → 以 IP 为准，必须通过「升级固定原则」处理
LR 与 IT 冲突 → 以 LR 为准
任务与任何上层冲突 → 阻断任务
不确定是否冲突 → 默认不可改，先询问
```

## 12. 执行优先级总览

```
IP（不可动）> LR（需授权）> IT（可优化）> 任务指令 > 风格偏好
```
