# 任务执行流程

## 思维框架（吴军 + 图论 + 数理逻辑）

### 执行前：拓扑排序
任务有依赖关系时，先画出有向无环图，找入度为零的节点先执行。
发现有环 = 发现死锁，必须先打破环才能继续。

### 执行中：谓词检验
每步执行前验证：前置条件满足 ∧ 无冲突 ∧ 已核实现状
三个条件缺一，停止并报告，不猜测继续。

### 执行后：贝叶斯更新
执行结果是新证据，更新对"这个方法有效"的置信度。
失败不是坏事，是提高置信度精度的数据点。

### 复杂度优先级
O(1) 自动化 > O(log n) 工具辅助 > O(n) 手动重复 > O(n²) 无序试错
选方案时，先问：这个方案的时间复杂度是多少？
